## ⚡ 5分で完了するデプロイ手順

### Step 1️⃣: GitHub に登録（2分）

```
1. https://github.com/signup
2. メール、パスワード、ユーザー名を入力
3. メール確認 → リンククリック
4. ✅ GitHub アカウント完成
```

### Step 2️⃣: Streamlit Cloud に登録（1分）

```
1. https://share.streamlit.io
2. 「Sign up」をクリック
3. 「Sign in with GitHub」をクリック
4. ✅ Streamlit Cloud アカウント完成
```

### Step 3️⃣: リポジトリを作成（1分）

```
1. https://github.com/new
2. Repository name: dashboard-app
3. Public を選択
4. 「Create repository」をクリック
```

### Step 4️⃣: ファイルをアップロード（1分）

```
1. GitHub の「Add file」→「Upload files」
2. すべてのファイルをドラッグ&ドロップ
   - app.py
   - requirements.txt
   - .streamlit/config.toml
   - 野菜_仕入れ数量表.xlsx
   - 野菜_販売数量表.xlsx
3. 「Commit changes」をクリック
```

### Step 5️⃣: Streamlit にデプロイ（自動）

```
1. https://share.streamlit.io
2. 「Create app」をクリック
3. 設定:
   - Repository: (ユーザー名)/dashboard-app
   - Branch: main
   - Main file path: app.py
4. 「Deploy」をクリック
5. ⏳ 1-2分待機...
6. ✅ デプロイ完了！
```

### Step 6️⃣: URL をシェア

```
生成された URL:
https://dashboard-yamada.streamlit.app/

👉 この URL をチーム内で共有！
```

## 🛡️ セキュリティ設定

### リポジトリを Private にする（本番環境向け）

```
1. GitHub → Settings → Danger Zone
2. 「Make private」をクリック
3. リポジトリ名を入力して確認
```

### Streamlit を Private にする（アクセス制限）

```
1. Streamlit Cloud → Settings
2. Visibility → Private
3. アクセス許可ユーザーを指定
```

## 🆘 トラブルシューティング

### ❌ "Module not found" エラー

**原因:** requirements.txt がない

**解決:**
```bash
GitHub で requirements.txt をアップロード
```

### ❌ "app.py が見つからない" エラー

**原因:** dashboard_app.py のままになっている

**解決:**
```
GitHub で app.py にリネーム
または、デプロイ時に「Main file path」を dashboard_app.py に変更
```

### ❌ "データが見つかりません" エラー

**原因:** Excel ファイルが GitHub にない

**解決:**
```
GitHub に以下をアップロード:
- 野菜_仕入れ数量表.xlsx
- 野菜_販売数量表.xlsx
```

### ❌ アプリが遅い

**原因:** 初回起動は時間がかかる

**解決:**
```
1. 30秒待つ
2. F5 でリロード
3. ブラウザキャッシュクリア
```
