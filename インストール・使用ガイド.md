## 🔧 Step 1: Pythonのインストール確認

### 1-1. Pythonがインストール済みか確認

**Windows の場合:**
```
ウィンドウズキー + R を押す
→ `cmd` と入力してEnter
→ 以下を入力
```

**Mac / Linux の場合:**
```
ターミナルを開く
→ 以下を入力
```

**確認コマンド:**
```bash
python --version
```

**出力例:**
```
Python 3.10.5
```

✅ **3.7 以上なら OK**
❌ **"command not found" が出た場合は Python をインストール**

### 1-2. Pythonのインストール（未インストールの場合）

#### Windows の場合

1. https://www.python.org/downloads/ にアクセス
2. 「Download Python 3.11」（または最新版）をクリック
3. ダウンロードされたファイルを実行
4. **重要**: インストール時に「Add Python to PATH」にチェックを入れる ✅
5. Install Now をクリック
6. インストール完了後、コマンドプロンプトで確認：
   ```bash
   python --version
   ```

#### Mac の場合

```bash
# Homebrewを使う場合
brew install python3

# または https://www.python.org/downloads/ からダウンロード
```

#### Linux (Ubuntu/Debian) の場合

```bash
sudo apt update
sudo apt install python3 python3-pip
```

## 📦 Step 3: ライブラリのインストール

### 3-1. requirements.txt を確認

フォルダに `requirements.txt` が入っているか確認：
```
streamlit==1.28.1
pandas==2.0.3
plotly==5.17.0
openpyxl==3.1.2
```

### 3-2. インストールコマンド実行

コマンドプロンプト/ターミナルで以下を実行：

```bash
pip install -r requirements.txt
```

**出力例:**
```
Collecting streamlit==1.28.1
  Downloading streamlit-1.28.1-py2.py3-none-any.whl (...MB)
Installing collected packages: streamlit, pandas, plotly, openpyxl
Successfully installed streamlit-1.28.1 pandas-2.0.3 plotly-5.17.0 openpyxl-3.1.2
```

⏳ **初回は 2-5 分かかる場合があります**

### 3-3. インストール確認

```bash
pip list
```

以下が表示されれば OK：
```
streamlit       1.28.1
pandas          2.0.3
plotly          5.17.0
openpyxl        3.1.2
```

## 🚀 Step 4: アプリの起動

### 4-1. Streamlit アプリを起動

コマンドプロンプト/ターミナルで以下を実行：

```bash
streamlit run dashboard_app.py
```

### 4-2. 初回起動メッセージ

以下のようなメッセージが表示されます：

```
  You can now view your Streamlit app in your browser.

  Local URL: http://localhost:8501
  Network URL: http://192.168.1.xxx:8501
```

### 4-3. ブラウザが自動的に開く

- **自動で開かない場合**: `http://localhost:8501` にアクセス
- **ポート 8501 が使用中の場合**: 別のポートで起動
  ```bash
  streamlit run dashboard_app.py --server.port 8502
  ```

### 4-4. 終了方法

コマンドプロンプト/ターミナルで：
```
Ctrl + C
```

## 🎓 機能の詳しい使い方

### 機能 1️⃣: KPI サマリーを確認

**何ができるか:**
- 総仕入れ・販売・在庫をワンクリックで確認
- 利益率をチェック
- 未販売品目の数を確認

**見方:**
```
📈 KPI サマリー
─────────────────────────────────────
総仕入れ数量    | 2,500 kg
総販売数量      | 2,250 kg
総仕入れ金額    | ¥375,000
総販売金額      | ¥562,500
─────────────────────────────────────
現在の在庫量    | 250 kg
利益            | ¥187,500
利益率          | 33.3%
未販売品目数    | 1 種類
```

### 機能 2️⃣: グラフで分析

#### 📈 「仕入れ vs 販売」タブ

**左グラフ: 品目別比較**
- 各野菜について、仕入れ（青）vs 販売（緑）が一目瞭然
- 赤い棒が大きい品目 = 在庫過剰

**右グラフ: 日付別推移**
- 時系列で仕入れと販売の推移を確認
- トレンドを把握できます

#### 📊 「品目別分析」タブ

**左グラフ: ステータス分布（円グラフ）**
- 全体の何%が在庫過剰か、品切れか一目瞭然
- 色の説明：
  - 🔴 赤: 在庫過剰
  - 🟠 オレンジ: 品切れ
  - 🔵 水色: 未販売
  - 🟢 緑: ちょうど売れた

**右グラフ: 品目別在庫量（横棒グラフ）**
- 各品目の在庫量をランキング表示
- 在庫過剰（赤）か完売（緑）かで色分け

#### 💰 「金額分析」タブ

**左グラフ: 仕入れ金額 vs 販売金額**
- 金額ベースで比較
- 赤字になっている品目を発見

**右グラフ: 売上効率**
- (販売数量 / 仕入れ数量) × 100 = %
- **100% = 完売**
- **80% = 20% の売れ残り**
- **120% = 在庫不足（売上逃し）**

#### 📦 「在庫状況」タブ

- 日付による在庫量の変化を追跡
- 赤い点線（ゼロ）を基準に、上下の動き確認
- 在庫が減少傾向か増加傾向かを判断

### 機能 3️⃣: 詳細テーブルをフィルター・ソート

#### 📋 フィルター機能

**ステータスでフィルター:**
```
☑️ 在庫過剰
☑️ 品切れ
☑️ 未販売
☑️ ちょうど
```

👉 **使い方:**
1. フィルターしたいステータスにチェック
2. チェックを外すと、その行は非表示
3. 例：「在庫過剰」のみチェック → 在庫過剰の行だけ表示

**品名でフィルター:**
```
▼ 品名を選択
[トマト] [キュウリ] [ニンジン] [玉ねぎ] [レタス] [ナス]
```

👉 **使い方:**
1. 特定の野菜をクリック
2. その野菜のデータのみ表示
3. 複数選択可能

**ソート機能:**
```
▼ ソート: 日付
   • 日付
   • 品名
   • 仕入れ数量
   • 販売数量
   • 在庫量
   • 仕入れ金額
```

👉 **使い方:**
1. ドロップダウンから選択
2. テーブルが選択した順でソート
3. 「仕入れ数量」でソート → 仕入れが多い順に並び替え

#### 📊 テーブル内容

| 列 | 説明 | 例 |
|---|------|-----|
| **日付** | データの日付 | 2026-03-23 |
| **品番** | 商品の識別番号 | V001 |
| **品名** | 商品名 | トマト |
| **仕入れ数量** | 仕入れた量（kg） | 50 |
| **販売数量** | 販売した量（kg） | 45 |
| **在庫量** | 現在の在庫（kg） | 5 |
| **仕入れ単価** | 1kg あたりの仕入れ価格 | ¥100 |
| **販売単価** | 1kg あたりの販売価格 | ¥150 |
| **仕入れ金額** | 仕入れ数量 × 仕入れ単価 | ¥5,000 |
| **販売金額** | 販売数量 × 販売単価 | ¥6,750 |
| **ステータス** | 在庫状態 | 在庫過剰 |

## 🔄 カスタムデータを使用する

### ステップ 1: CSVファイルを準備

**仕入れデータ (purchase.csv):**
```csv
日付,品番,品名,仕入れ数量(kg),仕入れ単価(円/kg),仕入れ金額(円)
2026-03-23,P001,商品A,100,200,20000
2026-03-23,P002,商品B,50,300,15000
2026-03-24,P001,商品A,110,200,22000
```

**販売データ (sales.csv):**
```csv
日付,品番,品名,販売数量(kg),販売単価(円/kg),販売金額(円)
2026-03-23,P001,商品A,90,350,31500
2026-03-23,P002,商品B,45,400,18000
2026-03-24,P001,商品A,100,350,35000
```

**ポイント:**
- 列のヘッダーは必ず上記と一致させる
- 日付は「2026-03-23」形式
- 仕入れ数量は kg 単位
- 金額は「仕入れ数量 × 単価」

### ステップ 2: アプリで設定

1. **サイドバーで「CSVをアップロード」を選択**
   ```
   ☑️ CSVをアップロード
   ```

2. **「仕入れデータ」ファイルをアップロード**
   - 「ファイルを選択」をクリック
   - `purchase.csv` を選択

3. **「販売データ」ファイルをアップロード**
   - 「ファイルを選択」をクリック
   - `sales.csv` を選択

4. **自動的に分析実行**
   - アップロード直後に分析が開始
   - グラフやテーブルが更新

## 🆘 よくあるエラー対応

### エラー 1: "Module 'streamlit' has no attribute..."

**原因:** Streamlit がインストールされていない

**解決方法:**
```bash
pip install streamlit
```

### エラー 2: "No module named 'pandas'"

**原因:** pandas がインストールされていない

**解決方法:**
```bash
pip install pandas
```

### エラー 3: "FileNotFoundError: [Errno 2] No such file or directory"

**原因:** ファイルが見つからない

**解決方法:**
1. `dashboard_app.py`, `野菜_仕入れ数量表.xlsx`, `野菜_販売数量表.xlsx` が同じフォルダにあるか確認
2. ファイル名に日本語が含まれている場合、英字に変更
3. ファイルが完全にダウンロードされているか確認

### エラー 4: "Address already in use"

**原因:** ポート 8501 が別のアプリで使用中

**解決方法:**
```bash
# 別のポートで起動
streamlit run dashboard_app.py --server.port 8502
```

### エラー 5: グラフが表示されない

**原因:** インターネット接続がない、またはブラウザキャッシュ

**解決方法:**
1. インターネット接続を確認
2. ブラウザのキャッシュをクリア（Ctrl + Shift + Delete）
3. ページをリロード（F5）

### エラー 6: アップロードしたファイルが認識されない

**原因:** ファイル形式が異なる、またはヘッダーが違う

**解決方法:**
1. **CSV の場合:** 文字コードを UTF-8 に変更
   - Excel で開く → 別名保存 → CSV UTF-8

2. **ヘッダーを確認:**
   ```
   必須ヘッダー（完全一致）:
   • 日付
   • 品番
   • 品名
   • 仕入れ数量(kg) [販売データは 販売数量(kg)]
   • 仕入れ単価(円/kg) [販売データは 販売単価(円/kg)]
   • 仕入れ金額(円) [販売データは 販売金額(円)]
   ```

## 💡 Tips & コツ

### Tip 1: 複数のデータセットを比較
1. CSV をダウンロード
2. 別の月のデータで「CSVをアップロード」実行
3. KPI の変化を比較

### Tip 2: 定期的な監視
1. 毎日 `dashboard_app.py` を実行
2. KPI サマリーで異常をチェック
3. ステータスが「品切れ」に変わった品目は急いで発注

### Tip 3: レポート作成
1. 「詳細テーブル」で必要なデータをフィルター
2. 「Excel ダウンロード」
3. Excel で合計行や注釈を追加
4. 経営層にレポート提出

### Tip 4: パフォーマンス改善
- データが多い場合（10,000 行以上）
  1. 日付範囲を限定する
  2. 特定の品目のみフィルター
  3. データを分割してアップロード
